# 题 #2 技术方案：基于 LangGraph + MCP 的「矿权日报 Agent」

## 1. 项目目标

在 24 小时内实现一个基于 MCP 协议和 LangGraph 编排的「矿权日报 Agent」。

用户输入：

```text
给我生成一份关于 Pilbara 锂矿的今日简报
```

系统输出 Markdown 简报，内容包括：

1. 新闻摘要；
2. NI 43-101 / 矿权报告中的 Indicated / Inferred 资源量数据；
3. 锂 / 相关金属价格走势；
4. 风险提示；
5. 引用源链接。

交付内容：

* 3 个 MCP server；
* 1 个基于 LangGraph 的 Agent client；
* `mcp-config.json`；
* `RUN.md`；
* `docker-compose.yml`；
* 可运行 demo。

---

## 2. 总体架构

整体采用：

```text
MCP Server 工具层 + LangGraph Agent 编排层 + Markdown 报告生成层
```

架构如下：

```text
User Input
   ↓
LangGraph Agent Client
   ↓
┌──────────────────────────────────────────────┐
│                BriefingState                 │
│ query / project / commodity / news / price   │
│ resources / risks / report / errors          │
└──────────────────────────────────────────────┘
   ↓
┌───────────────┬────────────────┬───────────────┐
│ mining-news   │ mineral-pdf     │ lme-price     │
│ MCP Server    │ MCP Server      │ MCP Server    │
└───────────────┴────────────────┴───────────────┘
   ↓
LangGraph Report Renderer
   ↓
Markdown Briefing
```

MCP 负责工具标准化，LangGraph 负责任务流程编排。

---

## 3. 技术选型

| 模块         | 技术选型                                | 说明                             |
| ---------- | ----------------------------------- | ------------------------------ |
| Agent 编排   | **LangGraph StateGraph**            | 强制使用 LangGraph 显式建图            |
| MCP server | Python + FastMCP / MCP Python SDK   | 快速暴露工具                         |
| MCP client | Python MCP client                   | 在 LangGraph node 内调用 MCP tools |
| PDF 解析     | PyMuPDF + pdfplumber + LLM fallback | 先规则解析，失败再模型修复                  |
| 新闻采集       | feedparser + httpx + trafilatura    | RSS 检索 + 正文抽取                  |
| 价格行情       | httpx + pandas                      | 查询价格、计算趋势                      |
| 报告生成       | LLM + 模板约束                          | 生成 Markdown                    |
| 容器化        | Docker Compose                      | 5 分钟内跑通                        |
| 配置         | YAML / JSON                         | 管理矿权别名、PDF URL、MCP server      |

# 5. LangGraph Agent 编排设计

## 5.1 为什么用 LangGraph

本题不能只做简单脚本串联，因为题目要求是 Agent 应用，并且明确要求使用 LangGraph。

因此 Agent client 采用 `StateGraph` 建图，所有节点都围绕一个共享状态 `BriefingState` 读写数据。

LangGraph 在这里承担 5 个职责：

1. 明确日报生成流程；
2. 管理跨节点状态；
3. 控制 MCP 工具调用顺序；
4. 支持条件分支和降级；
5. 支持报告质检和重写循环。

---

## 5.2 BriefingState 状态定义

```python
from typing import TypedDict, Optional, List, Dict, Any

class BriefingState(TypedDict, total=False):
    # 原始输入
    user_query: str

    # 意图解析结果
    project: str
    company: str
    commodity: str
    report_type: str
    days: int

    # 配置解析结果
    aliases: List[str]
    pdf_urls: List[str]

    # MCP 工具结果
    news_items: List[Dict[str, Any]]
    articles: List[Dict[str, Any]]
    resources: Dict[str, Any]
    latest_price: Dict[str, Any]
    price_trend: Dict[str, Any]

    # 中间分析结果
    news_summary: str
    resource_summary: str
    price_summary: str
    risks: List[str]

    # 报告结果
    markdown: str
    report_path: str

    # 质量控制
    quality_score: float
    quality_issues: List[str]
    revise_count: int

    # 异常与降级
    errors: List[Dict[str, Any]]
    warnings: List[str]
```


### 2. `resolve_asset`

作用：根据项目别名配置补全矿权信息。


作用：调用 `mining-news-mcp.search(query, days)`。

查询策略：

如果结果为空，进入条件分支：


### 7. `analyze_context`

作用：将新闻、资源量、价格走势转成结构化摘要。

输出：

```json
{
  "news_summary": "今日 Pilbara 相关新闻主要集中在扩产、销售与锂价预期。",
  "resource_summary": "报告显示 Indicated 和 Inferred 资源量均已披露，资源量数据置信度较高。",
  "price_summary": "锂价格近 30 天上涨 7.14%，短期趋势偏强。"
}
```

---

### 8. `generate_risks`

作用：基于新闻、资源量和价格生成风险提示。

风险类型：

| 风险类型  | 判断依据                                 |
| ----- | ------------------------------------ |
| 价格风险  | 价格 30 日涨跌幅、波动                        |
| 政策风险  | 新闻中出现 regulation / export / approval |
| 资源量风险 | PDF 抽取置信度低或报告较旧                      |
| 运营风险  | 新闻中出现 delay / production / shipment  |
| 市场风险  | demand / inventory / supply 变化       |

输出：

```json
{
  "risks": [
    "锂价短期波动可能影响收入预期。",
    "若资源量数据来自历史报告，需要关注最新公告是否修订。",
    "扩产计划仍需关注审批、资金和投产节奏。"
  ]
}
```

---

### 9. `render_report`

作用：生成 Markdown 日报。

输入：

* `articles`
* `resources`
* `latest_price`
* `price_trend`
* `risks`

输出：

```json
{
  "markdown": "# Pilbara 锂矿日报\n\n..."
}
```

---

### 10. `quality_check`

作用：检查 Markdown 是否满足交付要求。

检查项：

| 检查项         | 规则                     |
| ----------- | ---------------------- |
| 新闻摘要        | 至少 1 条新闻               |
| 储量数据        | 有 Indicated 或 Inferred |
| 价格走势        | 有 latest price 或 trend |
| 风险提示        | 至少 3 条                 |
| 引用源链接       | 新闻、PDF、价格至少各有来源        |
| Markdown 格式 | 标题、表格、列表完整             |

输出：

```json
{
  "quality_score": 8.5,
  "quality_issues": []
}
```

如果分数低于 8，进入 `revise_report`。

---

### 11. `revise_report`

作用：根据 `quality_issues` 修订报告。

最多重写 2 次。

重点修复：

* 缺引用；
* 风险提示不足；
* 表格字段缺失；
* 新闻摘要过短；
* 价格解释不清楚。

---

### 12. `save_report`

作用：保存 Markdown 文件。

输出路径：

```text
reports/pilbara_daily_briefing_2026-07-09.md
```

```text

  ↓
```

```text

  ↓
route_quality
    ├── pass → save_report
```

# 7. LangGraph 代码骨架

## 7.1 graph.py

```python
from langgraph.graph import StateGraph, START, END

from agent.state import BriefingState
from agent.nodes import (
    parse_intent,
    resolve_asset,
    search_news,
    expand_news_days,
    fetch_articles,
    extract_resources,
    fetch_price,
    analyze_context,
    generate_risks,
    render_report,
    quality_check,
    revise_report,
    save_report,
    save_report_with_warnings,
)

from agent.routers import (
    route_news_result,
    route_quality_result,
)

def build_graph():
    graph = StateGraph(BriefingState)

    graph.add_node("parse_intent", parse_intent)
    graph.add_node("resolve_asset", resolve_asset)
    graph.add_node("search_news", search_news)
    graph.add_node("expand_news_days", expand_news_days)
    graph.add_node("fetch_articles", fetch_articles)
    graph.add_node("extract_resources", extract_resources)
    graph.add_node("fetch_price", fetch_price)
    graph.add_node("analyze_context", analyze_context)
    graph.add_node("generate_risks", generate_risks)
    graph.add_node("render_report", render_report)
    graph.add_node("quality_check", quality_check)
    graph.add_node("revise_report", revise_report)
    graph.add_node("save_report", save_report)
    graph.add_node("save_report_with_warnings", save_report_with_warnings)

    graph.add_edge(START, "parse_intent")
    graph.add_edge("parse_intent", "resolve_asset")
    graph.add_edge("resolve_asset", "search_news")

    graph.add_conditional_edges(
        "search_news",
        route_news_result,
        {
            "enough_news": "fetch_articles",
            "expand_days": "expand_news_days",
            "continue_with_warning": "fetch_articles",
        }
    )

    graph.add_edge("expand_news_days", "search_news")
    graph.add_edge("fetch_articles", "extract_resources")
    graph.add_edge("extract_resources", "fetch_price")
    graph.add_edge("fetch_price", "analyze_context")
    graph.add_edge("analyze_context", "generate_risks")
    graph.add_edge("generate_risks", "render_report")
    graph.add_edge("render_report", "quality_check")

    graph.add_conditional_edges(
        "quality_check",
        route_quality_result,
        {
            "pass": "save_report",
            "revise": "revise_report",
            "save_with_warnings": "save_report_with_warnings",
        }
    )

    graph.add_edge("revise_report", "quality_check")
    graph.add_edge("save_report", END)
    graph.add_edge("save_report_with_warnings", END)

    return graph.compile()
```

## 7.2 routers.py

```python
def route_news_result(state):
    news_items = state.get("news_items", [])
    days = state.get("days", 1)

    if len(news_items) >= 1:
        return "enough_news"

    if days < 30:
        return "expand_days"

    return "continue_with_warning"


def route_quality_result(state):
    score = state.get("quality_score", 0)
    revise_count = state.get("revise_count", 0)

    if score >= 8:
        return "pass"

    if revise_count < 2:
        return "revise"

    return "save_with_warnings"
```

## 7.3 nodes.py 核心示例

```python
from datetime import date
from agent.mcp_clients import (
    call_mining_news_tool,
    call_mineral_pdf_tool,
    call_price_tool,
)
from agent.asset_resolver import resolve_asset_by_query
from agent.report_generator import generate_markdown_report
from agent.quality import check_report_quality


def parse_intent(state):
    query = state["user_query"]

    # 24h demo 版本可以先用规则 + LLM 兜底
    if "Pilbara" in query or "pilbara" in query.lower():
        return {
            "project": "Pilbara",
            "company": "Pilbara Minerals",
            "commodity": "lithium",
            "report_type": "daily_briefing",
            "days": 1,
            "revise_count": 0,
            "errors": [],
            "warnings": [],
        }

    return {
        "project": "unknown",
        "commodity": "unknown",
        "days": 1,
        "revise_count": 0,
        "errors": [],
        "warnings": ["未能明确识别矿权项目，后续会尝试模糊匹配。"],
    }


def resolve_asset(state):
    asset = resolve_asset_by_query(
        project=state.get("project"),
        company=state.get("company"),
        commodity=state.get("commodity"),
    )

    return {
        "aliases": asset.get("aliases", []),
        "pdf_urls": asset.get("pdf_urls", []),
        "company": asset.get("company", state.get("company")),
        "commodity": asset.get("commodity", state.get("commodity")),
    }


async def search_news(state):
    query = " ".join([
        state.get("project", ""),
        state.get("company", ""),
        state.get("commodity", "")
    ]).strip()

    try:
        result = await call_mining_news_tool(
            tool_name="search",
            arguments={
                "query": query,
                "days": state.get("days", 1)
            }
        )
        return {"news_items": result.get("items", [])}
    except Exception as e:
        return {
            "news_items": [],
            "errors": state.get("errors", []) + [{
                "node": "search_news",
                "error": str(e)
            }]
        }


def expand_news_days(state):
    current_days = state.get("days", 1)

    if current_days <= 1:
        next_days = 7
    elif current_days <= 7:
        next_days = 30
    else:
        next_days = 30

    return {"days": next_days}


async def fetch_articles(state):
    articles = []

    for item in state.get("news_items", [])[:3]:
        url = item.get("url")
        if not url:
            continue

        try:
            article = await call_mining_news_tool(
                tool_name="fetch_article",
                arguments={"url": url}
            )
            articles.append(article)
        except Exception as e:
            articles.append({
                "title": item.get("title"),
                "url": url,
                "content": item.get("summary", ""),
                "warning": f"正文抓取失败，使用摘要代替：{str(e)}"
            })

    return {"articles": articles}


async def extract_resources(state):
    pdf_urls = state.get("pdf_urls", [])
    warnings = state.get("warnings", [])

    if not pdf_urls:
        return {
            "resources": {},
            "warnings": warnings + ["未配置资源量 PDF，储量数据部分待人工补充。"]
        }

    for pdf_url in pdf_urls:
        try:
            result = await call_mineral_pdf_tool(
                tool_name="extract_resources",
                arguments={"pdf_url": pdf_url}
            )

            if result.get("resources"):
                return {"resources": result}

        except Exception as e:
            warnings.append(f"PDF 抽取失败：{pdf_url}，错误：{str(e)}")

    return {
        "resources": {},
        "warnings": warnings + ["所有 PDF 均抽取失败，资源量数据待人工复核。"]
    }


async def fetch_price(state):
    commodity = state.get("commodity", "lithium")
    today = date.today().isoformat()

    latest_price = {}
    price_trend = {}
    warnings = state.get("warnings", [])

    try:
        latest_price = await call_price_tool(
            tool_name="get_price",
            arguments={
                "commodity": commodity,
                "date": today
            }
        )
    except Exception as e:
        warnings.append(f"价格查询失败：{str(e)}")

    try:
        price_trend = await call_price_tool(
            tool_name="get_trend",
            arguments={
                "commodity": commodity,
                "days": 30
            }
        )
    except Exception as e:
        warnings.append(f"价格趋势查询失败：{str(e)}")

    return {
        "latest_price": latest_price,
        "price_trend": price_trend,
        "warnings": warnings,
    }


def analyze_context(state):
    articles = state.get("articles", [])
    resources = state.get("resources", {})
    price_trend = state.get("price_trend", {})

    news_summary = "暂无相关新闻。"
    if articles:
        news_summary = f"共检索到 {len(articles)} 条相关新闻，主要围绕项目进展、市场价格和运营风险。"

    resource_summary = "资源量数据待人工复核。"
    if resources.get("resources"):
        resource_summary = "已抽取到 Indicated / Inferred 资源量数据，可进入日报。"

    price_summary = "价格趋势数据不可用。"
    if price_trend:
        price_summary = (
            f"近 {price_trend.get('days', 30)} 天价格趋势为 "
            f"{price_trend.get('trend', 'unknown')}，"
            f"涨跌幅 {price_trend.get('change_pct', 'N/A')}%。"
        )

    return {
        "news_summary": news_summary,
        "resource_summary": resource_summary,
        "price_summary": price_summary,
    }


def generate_risks(state):
    risks = []

    price_trend = state.get("price_trend", {})
    resources = state.get("resources", {})
    articles = state.get("articles", [])

    if price_trend.get("change_pct") is not None:
        risks.append("价格短期波动可能影响项目收入预期。")

    if not resources.get("resources"):
        risks.append("资源量数据未能稳定抽取，需人工复核后再用于正式投资判断。")

    if articles:
        risks.append("新闻事件可能影响项目投产节奏、审批进度或市场预期。")

    risks.append("矿权项目仍需关注政策、环保、运输和融资等外部变量。")

    return {"risks": risks}


def render_report(state):
    markdown = generate_markdown_report(state)
    return {"markdown": markdown}


def quality_check(state):
    result = check_report_quality(state.get("markdown", ""), state)
    return {
        "quality_score": result["score"],
        "quality_issues": result["issues"],
    }


def revise_report(state):
    current = state.get("markdown", "")
    issues = state.get("quality_issues", [])
    revise_count = state.get("revise_count", 0)

    revised = generate_markdown_report(
        state,
        previous_markdown=current,
        quality_issues=issues,
    )

    return {
        "markdown": revised,
        "revise_count": revise_count + 1,
    }
```

````
# 17. 关键设计取舍

## 17.1 不建议让 LLM 自由决定工具调用

本题是日报生成，不是开放式问答。为了保证 24 小时内可交付，Agent 流程应该相对确定。

推荐方式：

```text
LangGraph 显式流程控制 + LLM 局部生成摘要
````

不推荐：

```text
LLM 自由 ReAct 选择工具
```

原因：

1. 显式流程更容易验收；
2. MCP 工具调用顺序更可控；
3. 报告结构更稳定；
4. 出错后更容易定位；
5. 可以清晰展示 LangGraph 编排能力。

---

## 17.2 MCP Server 做薄，LangGraph 做编排

MCP server 只负责单一能力：

* 新闻 MCP：搜索、正文抓取；
* PDF MCP：资源量抽取；
* 价格 MCP：价格、趋势。

LangGraph 负责：

* 先调什么；
* 后调什么；
* 失败怎么降级；
* 报告怎么生成；
* 报告不合格怎么修订。

---

## 17.3 质检循环要简单

`quality_check` 不需要做复杂模型评估，24h 版本只做规则 + LLM 轻量判断：

规则检查：

1. 是否有标题；
2. 是否有新闻摘要；
3. 是否有资源量表格；
4. 是否有价格走势；
5. 是否有风险提示；
6. 是否有引用链接。

LLM 判断：

1. 是否像一份完整日报；
2. 是否有明显缺项；
3. 是否有事实和引用不对应的问题。

# 18. 最终验收标准

必须满足：

1. 有 3 个 MCP server；
2. 每个 server 暴露题目要求的 tools；
3. Agent client 必须基于 LangGraph；
4. LangGraph 中能看到明确 nodes / edges / conditional edges；
5. 输入一句自然语言，能输出 Markdown 简报；
6. Markdown 包含新闻、资源量、价格走势、风险提示和引用；
7. 有 `mcp-config.json`；
8. 有 `RUN.md`；
9. 有 `docker-compose.yml`；
10. 5 分钟内可以跑起来。

加分项：

1. `graph.get_graph().draw_mermaid_png()` 输出流程图；
2. `quality_check → revise_report` 有真实循环；
3. 失败降级信息写入 warnings；
4. 报告保存到 `reports/`；
5. 每个 MCP 工具都有独立测试；
6. LangGraph 状态可打印，便于评审展示。
