# 我的技术方案 — 矿权日报 Agent

> ：基于 **LangGraph + MCP 协议**的矿权日报生成 Agent，输入一句自然语言，自动采集新闻、解析 PDF 资源量、查询价格行情，输出结构化 Markdown 日报，并提供实时进度可视化的 Web 界面。

---

## 一、项目背景与目标

矿权投资研究需要每日整合多源信息（新闻、NI 43-101 资源量报告、金属价格），人工汇总耗时且易遗漏。本项目用 Agent 自动化这一流程：

**输入**：`给我生成一份关于 Pilbara 锂矿的今日简报`

**输出**：包含新闻摘要、资源量数据、价格走势、风险提示、引用源的 Markdown 日报。

---

## 二、整体架构

```text
┌─────────────────────────────────────────────────────────┐
│                     React Web 前端                       │
│        (输入指令 / 实时进度 / 报告展示 / 历史记录)          │
└──────────────────────┬──────────────────────────────────┘
                       │ REST + SSE
┌──────────────────────▼──────────────────────────────────┐
│                  FastAPI 后端服务                         │
│         (任务管理 / SSE 事件流 / 报告存储)                 │
└──────────────────────┬──────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────┐
│              LangGraph Agent 编排层                      │
│     14 节点 StateGraph + 2 条件路由 + 质检循环            │
└──────────────────────┬──────────────────────────────────┘
                       │ MCP 协议 (stdio)
┌───────────┬──────────┴──────────┬───────────────────────┐
│           │                     │
▼           ▼                     ▼
mining-news  mineral-pdf          lme-price
MCP Server   MCP Server           MCP Server
(新闻采集)    (PDF资源量抽取)       (价格行情)
```

**三层分工**：
- **MCP 工具层**：3 个独立 Server，标准化封装数据采集能力，可被任意 MCP Client 复用
- **LangGraph 编排层**：显式 StateGraph，控制任务流转、条件分支、质检循环
- **Web 服务层**：FastAPI 提供 REST + SSE，React 前端实时可视化

## 三、核心技术选型

| 层 | 技术 | 选型理由 |
|----|------|----------|
| Agent 编排 | **LangGraph StateGraph** | 显式建图，流程可观测，支持条件路由和循环，比 ReAct 更可控 |
| 工具协议 | **MCP (Model Context Protocol)** | 工具标准化，Server 可被 Claude Desktop / Cursor 等复用 |
| MCP SDK | **FastMCP** | 装饰器风格，开发效率高，`@mcp.tool` 一行暴露工具 |
| LLM | **通义千问 Qwen**（OpenAI 兼容接口） | 国产模型，DashScope 兼容 OpenAI SDK，切换成本低 |
| 后端 | **FastAPI + SSE** | 异步高性能，原生支持 StreamingResponse 实现 SSE |
| 前端 | **React 18 + TypeScript + Vite + Tailwind + Zustand** | 组件化 + 类型安全 + 轻量状态管理 |
| PDF 解析 | **PyMuPDF + pdfplumber** | 规则解析表格，失败降级 Mock |
| 容器化 | **Docker Compose** | 前后端一键部署 |

## 四、LangGraph 工作流设计

14 个节点构成完整日报生成流水线，含 2 处条件路由：

```text
START
  → parse_intent        解析用户意图（LLM / 规则）
  → resolve_asset       匹配矿权资产配置
  → search_news         搜索新闻
  → [路由1] ─────────────────────────────────┐
      ├─ enough_news(≥3条) → fetch_articles   │
      ├─ expand_days      → expand_news_days  │ 扩大天数后重搜
      └─ continue_warning → fetch_articles    │ 带警告继续
  → fetch_articles      抓取新闻正文
  → extract_resources   抽取 PDF 资源量
  → fetch_price         获取价格行情
  → analyze_context     聚合上下文
  → generate_risks      生成风险提示（LLM / 规则）
  → render_report       渲染 Markdown
  → quality_check       质量检查
  → [路由2] ─────────────────────────────────┐
      ├─ pass           → save_report         │
      ├─ fail(retry<2)  → revise_report → quality_check (循环)
      └─ fail(retry=2)  → save_report_with_warnings
  → END
```

**关键设计**：
- **新闻搜索降级**：无结果时自动扩大搜索天数（1→7→30 天）
- **质检循环**：不合格报告最多修订 2 次，仍不合格带警告保存（保证有产出）
- **LLM 可选**：未配置 API Key 时纯规则模式运行，保证零依赖可 demo

## 五、MCP Server 设计

3 个 Server 通过 stdio 传输，由 Agent 自动拉起子进程：

| Server | 工具 | 输入 | 输出 |
|--------|------|------|------|
| mining-news-mcp | `search(query, days)` | 关键词 + 天数 | 新闻列表（标题/URL/摘要/来源） |
| | `fetch_article(url)` | 文章 URL | 正文 + 引用源 |
| mineral-pdf-mcp | `extract_resources(pdf_url)` | PDF URL | 资源量（类别/矿石量/品位/金属量） |
| lme-price-mcp | `get_price(commodity)` | 矿种 | 最新价 + 涨跌 |
| | `get_trend(commodity, days)` | 矿种 + 天数 | MA7/MA30/趋势/观测值 |

**降级策略**：每个 Server 真实数据源失败时自动降级到 Mock 数据（基于 Pilbara 锂矿真实公开数据模拟），保证 demo 任何时候可运行。降级信息写入 `warnings` 字段，前端展示警告。

## 六、关键技术亮点

### 1. LangGraph 双流模式（解决重复执行问题）

后端通过 `graph.astream(stream_mode=["updates", "values"])` 一次遍历同时获取：
- `updates` 流：每个节点的状态增量 → 转为 SSE 事件推给前端
- `values` 流：完整状态快照 → 作为最终结果

避免了"先 astream 推事件、再 ainvoke 取结果"导致图执行两遍的问题，执行时间从 ~115s 降至 ~57s。

### 2. SSE 实时进度可视化

前端通过 `EventSource` 订阅节点事件，14 个节点的执行状态实时渲染为时间线（pending → running → success/warning/skipped），用户能清楚看到 Agent "在做什么"。

### 3. 质检循环保障产出质量

规则检查 + LLM 质检双层把关：
- 规则层：必填字段、数据完整性、格式规范
- LLM 层：内容连贯性、风险覆盖度（可选）
- 不合格自动修订，最多 2 次循环，兜底带警告保存

### 4. LLM 零配置可运行

`DASHSCOPE_API_KEY` 未配置时，意图解析、摘要生成、风险判断全部走规则模板，系统功能完整。配置后自动增强为 LLM 生成，平滑升级。

## 七、前端设计

| 页面 | 功能 |
|------|------|
| 首页 | 指令输入 + 示例引导 |
| 执行中页 | 左侧 14 节点进度时间线 + 右侧实时运行日志 |
| 报告详情页 | Markdown 报告 + 新闻/资源量/价格/风险结构化展示 + 下载 |
| 历史记录页 | 报告列表 + 搜索 + 状态筛选 |

**UI 风格**：暗紫主题（#0a0a0f 背景 / #8b5cf6 强调色）+ 毛玻璃卡片（backdrop-blur）。

**状态管理**：Zustand 管理任务状态、节点进度、事件日志；自定义 Hooks 封装 API 调用、SSE 订阅、报告下载。
